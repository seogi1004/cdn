학술적 근거 및 업계 동향 검증 (Expanded)

1. 인지과학 (Miller's Law)

Miller의 법칙(7±2)에 따르면 인간의 단기 기억 용량은 제한적입니다. 수십 개의 노드가 얽힌 MSA Topology는 인간의 인지 한계를 즉시 초과하므로 AI의 '패턴 인식' 보조가 필수적입니다.

* 반면 Line 차트는 'Gestalt 법칙'에 의해 직관적 인식이 가능하여 AI 효용이 낮음.

2. 경쟁사 기술 (Trend)

  • Dynatrace (Davis AI): 단순 상관관계가 아닌 'Smartscape' 토폴로지 기반의 인과관계(Causality) 분석에 집중.
  • Datadog (Watchdog): 설정 없이 이상 징후를 찾는 Auto-Detection에 주력. (당사의 Future Work 방향과 일치)

3. LLM 최신 연구

LLM은 단순 시계열 수치보다 구조적 데이터(Graph/Tree)가 주어졌을 때 할루시네이션이 감소하고 추론 능력이 향상됨이 입증되었습니다.

→ Tier 1(Topology) 집중 전략의 기술적 타당성 확보.

Future Work: 대시보드 리플레이 + AI Auto-Detect Technical Roadmap

패러다임 전환: "Replay & Chain Analysis"

단일 스냅샷 분석의 한계를 넘어, 과거 시점을 재생(Replay)하며 시간 순서에 따른 인과관계(Causality)를 분석합니다. 장애 인지 시간(MTTD)을 획기적으로 단축하는 것이 목표입니다.

⚙️ 기술적 구현 계획 (Technical Specification)

1. Web Worker 아키텍처

UI 블로킹 방지를 위해 백그라운드 분석 워커(`instanceDataWorker.ts` 유사 구조) 활용. ANOMALY_DETECTED 이벤트 발행 구조.

2. 분석 주기 최적화 (이원화)

- 실시간: 임계값 기반 경량 탐지 (매 프레임)
- Deep 분석: 이상 감지 시 LLM 호출 (60초 주기)

3. 비용 효율화 (Tiered Analysis)

Browser LLM(Phi-3/Gemini Nano)으로 1차 필터링 후, 필요한 경우만 Server LLM 호출.

4. 구현 로드맵 (Phases)

  • Phase 1: 과거 인스턴스 조회 구현 (PRD 진행중)
  • Phase 2: 자동 트리거 및 Worker 개발
  • Phase 3: 연쇄 장애 패턴 템플릿화
